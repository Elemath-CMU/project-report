ในการจัดทำโครงงานสื่อการสอนคณิตศาสตร์ประเภท Sandbox สำหรับนักเรียนชั้นประถมปลาย คณะผู้จัดทำได้ศึกษาทฤษฎี แนวคิด และเทคโนโลยีต่างๆ ที่เกี่ยวข้อง

\section{Fraction models}
การทำความเข้าใจเรื่องเศษส่วนถือเป็นพื้นฐานสำคัญทางคณิตศาสตร์ แต่เนื่องจากเศษส่วนมีลักษณะเป็นนามธรรมสูง ผู้เรียนจึงมักประสบปัญหาในการทำความเข้าใจ 
การใช้แบบจำลองทางคณิตศาสตร์ (Mathematical Models) จึงเป็นเครื่องมือสำคัญที่ช่วยให้ผู้เรียนมองเห็นภาพรวมได้อย่างเป็นรูปธรรม ซึ่งแบบจำลองเศษส่วนแบ่งออกเป็น 3 ประเภทหลัก ได้แก่

\subsection{Area model}
เป็นการแสดงเศษส่วนโดยแบ่งพื้นที่ของรูปเรขาคณิต (เช่น วงกลม หรือ สี่เหลี่ยม) ออกเป็นส่วนๆ ที่มีขนาดเท่ากัน เป็นแบบจำลองที่พบได้บ่อยที่สุดในหนังสือเรียนทั่วไป
\\\hspace*{2em}อย่างไรก็ตาม Area model นั้นมีข้อจำกัดในเรื่องของการสื่อความหมายของเศษส่วนที่มีค่ามากกว่า 1 หรือเศษเกิน ซึ่งอาจทำให้เด็กเกิดความสับสนได้เมื่อต้องเรียนรู้เนื้อหาของเศษเกินนั่นเอง

\subsection{Set model}
เป็นการแสดงเศษส่วนโดยใช้กลุ่มของวัตถุ โดยพิจารณาจำนวนวัตถุที่สนใจเปรียบเทียบกับจำนวนวัตถุทั้งหมดในกลุ่ม
\\\hspace*{2em}อย่างไรก็ตาม Set model นั้นทำให้เกิดความสับสนได้หากมีการคำนวนเข้ามาเกี่ยวข้อง เช่น 
การบวกลบเศษส่วนที่มีตัวส่วนไม่เท่ากัน เพราะ Set model นั้นไม่สามารถแสดงความสัมพันธ์ระหว่างเศษส่วนได้อย่างชัดเจนนั่นเอง

\subsection{Length model}
เป็นการแสดงปริมาณของเศษส่วนโดยอาศัยความยาวหรือระยะทาง มักนำเสนอผ่านรูปแบบของเส้นจำนวน (Number Line) หรือแถบความยาว (Fraction Strips) งานวิจัยทางการศึกษาพบว่า length model 
เป็นแบบจำลองในการช่วยให้ผู้เรียนทำความเข้าใจการเปรียบเทียบเศษส่วนได้ง่ายกว่าแบบอื่น เนื่องจากผู้เรียนสามารถนำความยาวมาเปรียบเทียบกันได้โดยตรง\cite{ejmste}
\\\hspace*{2em}อย่างไรก็ตามการเรียนรู้ด้วย Length model นั้นจำเป็นที่จะต้องมีความเข้าใจพื้นฐานเกี่ยวกับการวัดความยาวและหน่วยวัดต่าง ๆ มาก่อน จึงอาจทำให้เด็กที่ยังไม่มีพื้นฐานเหล่านี้เกิดความสับสนได้

\subsection {Software Development Life Cycle (SDLC)}
    โดย SDLC ที่เลือกใช้ในโครงงานนี้คือแบบ Agile Model ซึ่งเป็นกระบวนการพัฒนาซอฟต์แวร์แบบทำไปปรับไป โดยจะมีการปรับปรุงและพัฒนาระบบอย่างต่อเนื่องตามความต้องการของผู้ใช้ โดยในแต่ละรอบของการพัฒนาจะมีการออกแบบ การพัฒนา การทดสอบ การส่งมอบระบบ และทบทวนสิ่งที่ทำไปในรอบนั้นๆเพื่อปรับปรุงในรอบถัดไป
    \\\hspace*{2em}เนื่องจากโครงงานนี้มีความต้องการที่อาจเปลี่ยนแปลงได้ตลอดเวลา และต้องการการตอบสนองที่รวดเร็วต่อความต้องการของผู้ใช้ Agile Model จึงเป็นตัวเลือกที่เหมาะสม
    ซึ่งในขั้นตอนแต่ละขั้นตอนของ Agile Model ที่กลุ่มของเราใช้นั้นมีรายละเอียดดังนี้
    \begin{itemize}
        \item Planning - วางแผนการพัฒนาและกำหนดความต้องการของระบบ
        \item Design - ออกแบบระบบตามความต้องการที่ได้จากขั้นตอน Planning
        \item Development - พัฒนาระบบตามแบบที่ได้ออกแบบไว้
        \item Testing - ทดสอบระบบเพื่อหาข้อผิดพลาดและปรับปรุงระบบ
        \item Deployment - การให้ผู้ใช้ทดลองใช้งานระบบ
        \item Review - ทบทวนสิ่งที่ทำไปในรอบนั้นๆ เพื่อปรับปรุงในรอบถัดไป
    \end{itemize}
