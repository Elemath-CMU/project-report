\section{Fraction models}
การทำความเข้าใจเรื่องเศษส่วนถือเป็นพื้นฐานสำคัญทางคณิตศาสตร์ แต่เนื่องจากเศษส่วนมีลักษณะเป็นนามธรรมสูง ผู้เรียนจึงมักประสบปัญหาในการทำความเข้าใจ 
การใช้แบบจำลองทางคณิตศาสตร์ (Mathematical models) จึงเป็นเครื่องมือสำคัญที่ช่วยให้ผู้เรียนมองเห็นภาพรวมได้อย่างเป็นรูปธรรม ซึ่งแบบจำลองเศษส่วนแบ่งออกเป็น 3 ประเภทหลัก ได้แก่

\subsection{Area model}
เป็นการแสดงเศษส่วนโดยแบ่งพื้นที่ของรูปเรขาคณิต (เช่น วงกลม หรือ สี่เหลี่ยม) ออกเป็นส่วน ๆ ที่มีขนาดเท่ากัน เป็นแบบจำลองที่พบได้บ่อยที่สุดในหนังสือเรียนทั่วไป
\\\hspace*{2em}อย่างไรก็ตาม area model นั้นมีข้อจำกัดในเรื่องของการสื่อความหมายของเศษส่วนที่มีค่ามากกว่า 1 หรือเศษเกิน ซึ่งอาจทำให้เด็กเกิดความสับสนได้เมื่อต้องเรียนรู้เนื้อหาของเศษเกิน

\caption[Area model]{Area model of fraction $\frac{3}{4}$~\cite{tadeu2024fractions}}
\end{figure}

\subsection{Set model}
เป็นการแสดงเศษส่วนโดยใช้กลุ่มของวัตถุ โดยพิจารณาจำนวนวัตถุที่สนใจเปรียบเทียบกับจำนวนวัตถุทั้งหมดในกลุ่ม
\\\hspace*{2em}อย่างไรก็ตาม set model นั้นทำให้เกิดความสับสนได้หากมีการคำนวนเข้ามาเกี่ยวข้อง เช่น 
การบวกลบเศษส่วนที่มีตัวส่วนไม่เท่ากัน เพราะ set model นั้นไม่สามารถแสดงความสัมพันธ์ระหว่างเศษส่วนได้อย่างชัดเจน

\subsection{Length model}
เป็นการแสดงปริมาณของเศษส่วนโดยอาศัยความยาวหรือระยะทาง มักนำเสนอผ่านรูปแบบของเส้นจำนวน (number line) หรือแถบความยาว (fraction strips) งานวิจัยทางการศึกษาพบว่า length model 
เป็นแบบจำลองในการช่วยให้ผู้เรียนทำความเข้าใจการเปรียบเทียบเศษส่วนได้ง่ายกว่าแบบอื่น เนื่องจากผู้เรียนสามารถนำความยาวมาเปรียบเทียบกันได้โดยตรง\cite{tadeu2024fractions}
\\\hspace*{2em}อย่างไรก็ตามการเรียนรู้ด้วย length model นั้นจำเป็นที่จะต้องมีความเข้าใจพื้นฐานเกี่ยวกับการวัดความยาวและหน่วยวัดต่าง ๆ มาก่อน จึงอาจทำให้เด็กที่ยังไม่มีพื้นฐานเหล่านี้เกิดความสับสนได้

\section{Platform development tools}
    \subsection{TypeScript}
    TypeScript~\cite{typescript} คือภาษาโปรแกรมที่เป็น superset ของ JavaScript โดยมีการเพิ่มระบบ static typing หรือการระบุชนิดตัวแปรที่ชัดเจนลงไป
    โดย TypeScript มีคุณสมบัติสำคัญต่อการพัฒนาโครงงานนี้ในด้านต่าง ๆ ดังนี้
    \begin{itemize}
        \item Type safety: ช่วยลดข้อผิดพลาดในระหว่างการเขียนโค้ด เช่น การส่งค่าตัวเลขเศษส่วนผิดประเภท
        \item Readability and maintenance: โค้ดมีความเป็นระเบียบและอ่านง่ายขึ้น ช่วยให้สมาชิกในทีมที่เป็น developer สามารถทำงานร่วมกันได้อย่างมีประสิทธิภาพ
        \item Tooling support: สามารถใช้ทำงานร่วมกับ Visual Studio Code ได้
    \end{itemize}

    \subsection{Framework}
    Vite~\cite{vite} คือเครื่องมือที่ช่วยในการพัฒนา web application ซึ่งเหมาะสำหรับการพัฒนาโครงงานนี้ที่ต้องการการปรับปรุงและทดสอบระบบอย่างรวดเร็ว
    ทำให้โครงงานนี้มีการนำ Vite มาใช้ในการพัฒนา โดย Vite มีคุณสมบัติสำคัญต่อการพัฒนาโครงงานนี้ในด้านต่าง ๆ ดังนี้
    \begin{itemize}
        \item Fast development: Vite มีการใช้ ES modules ในการโหลดโค้ด ทำให้สามารถทำ hot module replacement (HMR) ได้อย่างรวดเร็ว
        \item Modern features: รองรับฟีเจอร์ใหม่ ๆ ของ JavaScript และ TypeScript ได้อย่างเต็มที่
        \item Simple configuration: มีการตั้งค่าที่ง่ายและไม่ซับซ้อน ทำให้ทีม developer สามารถเริ่มต้นการพัฒนาได้อย่างรวดเร็ว
    \end{itemize}

    \subsection{Deployment method}
    GitHub~\cite{GitHub} และ Vercel~\cite{vercel} เป็นแพลตฟอร์มที่ช่วยในการจัดการโค้ดและการ deploy web application โดยโครงงานนี้ได้มีการนำแพลตฟอร์มทั้งสองมาใช้ร่วมกันดังนี้
    \begin{itemize}
        \item GitHub: ช่วยให้ทีม developer สามารถทำงานร่วมกัน และจัดการโค้ดได้อย่างมีประสิทธิภาพผ่านการใช้ version control และ pull requests
        \item Vercel: ช่วยในการ deploy web application โดยสามารถเชื่อมต่อกับ GitHub เพื่อทำการ deploy อัตโนมัติเมื่อมีการ push โค้ดใหม่ขึ้นไป
    \end{itemize}

\subsection{งานวิจัยทางการศึกษาเกี่ยวกับการสอนเรื่องเศษส่วน}
    จากงานวิจัย~\cite{rojo_length_models} ที่คณะผู้จัดทำได้ศึกษาพบว่า ความเข้าใจเรื่องจำนวนเป็นเรื่องพื้นฐานที่สำคัญในการเรียนรู้คณิตศาสตร์ แต่นักเรียนมักจะมีความเข้าใจที่ไม่ถูกต้องเกี่ยวกับเศษส่วน ซึ่งส่งผลต่อการเรียนรู้ในระดับที่สูงขึ้น
    จึงได้มีการทดลองแบบ multiple probe design โดยทำการทดลองการสอนนักเรียน ซึ่งได้ผลสรุปออกมาว่านักเรียนมีความเข้าใจเกี่ยวกับเศษส่วนและการแปลงค่าทศนิยมที่ดีขึ้นหลังจากการสอนแบบใช้ length model
    ส่งผลให้คณะผู้จัดทำได้มีการนำแนวคิดของ length model มาใช้ในการออกแบบและพัฒนาระบบของโครงงานนี้

\subsection[ทฤษฎีเกมเพื่อการศึกษา]{ทฤษฎีเกมเพื่อการศึกษา (Gamification)}
    ทฤษฎีเกมเพื่อการศึกษา~\cite{Gamifi} คือการนำหลักการของทฤษฎีเกมมาใช้ในการออกแบบสื่อการสอน เพื่อเพิ่มความสนุกสนานและกระตุ้นความสนใจในการเรียนรู้เรื่องเศษส่วน โดยโครงงานนี้ได้มีการสร้างระบบตอน (episode) ที่ช่วยให้ผู้เรียนรู้สึกถึงความก้าวหน้าในการเรียนรู้
    และมีการใส่ระบบแผนที่ (map) ที่ช่วยให้ผู้เรียนสามารถเลือกเรียนรู้เนื้อหาในลำดับที่ต้องการได้ ซึ่งจะช่วยเพิ่มความยืดหยุ่นในการเรียนรู้ จากการที่สามารถเลือกเรียนรู้เนื้อหาในลำดับที่ต้องการได้ จะช่วยให้ผู้เรียนสามารถปรับเปลี่ยนเส้นทางการเรียนรู้ได้ตามความสนใจและความต้องการของตนเอง รวมไปถึงการกลับมาเล่นตอนที่เคยเล่นไปแล้วเพื่อทบทวนเนื้อหาได้อีกด้วย

\subsection {ทฤษฎีการวัดและการเปรียบเทียบ}
    การวัดและการเปรียบเทียบเป็นทฤษฎีที่สำคัญเมื่อใช้แบบจำลอง length model จากการที่เศษส่วนถูกนำเสนอผ่านความยาวหรือระยะทาง เพราะ การวัดจะช่วยให้ผู้เรียนสามารถเข้าใจแนวคิดของความยาวนั่นเอง
    โดยโครงงานนี้ได้มีการนำ unit iteration หรือหน่วยย่อย (unit fraction เช่น $\frac{1}{4}$) มาวางต่อกันจนเป็นหน่วยใหญ่ ซึ่งจะช่วยทำให้เด็กเข้าใจว่าเศษส่วนคือ "ปริมาณ" ที่วางบนเส้นจำนวนได้ ไม่ใช่แค่ตัวเลขสองตัวที่มีเส้นขีดกั้น
    นอกจากนี้การวัดยังจะช่วยให้เด็กสามารถเปรียบเทียบเศษส่วนได้ง่ายขึ้น เช่น การเปรียบเทียบ $\frac{1}{4}$ กับ $\frac{1}{2}$ โดยการวางหน่วยย่อย $\frac{1}{4}$ และ $\frac{1}{2}$ บนเส้นจำนวน ซึ่งจะเห็นได้ว่า $\frac{1}{2}$ มีความยาวมากกว่า $\frac{1}{4}$ นั่นเอง

\subsection[CPA approach (Concrete-Pictorial-Abstract)]{CPA approach (Concrete-Pictorial-Abstract)}
    CPA approach~\cite{cpa} เป็นแนวทางการสอนที่เน้นการเรียนรู้ผ่านสามขั้นตอน ได้แก่ ขั้นตอนที่เป็นรูปธรรม (concrete) ที่ผู้เรียนได้สัมผัสและปฏิบัติจริง ขั้นตอนที่เป็นภาพ (pictorial) ที่ผู้เรียนได้เห็นภาพแทนของแนวคิด และขั้นตอนที่เป็นนามธรรม (abstract) ที่ผู้เรียนได้เข้าใจแนวคิดในรูปแบบของสัญลักษณ์หรือสูตรคณิตศาสตร์
    โดยโครงงานนี้ได้มีการนำแนวทาง CPA มาใช้ในการออกแบบเนื้อหาและกิจกรรมการเรียนรู้ ด้วยการเป็นสื่อระหว่างขั้นตอนที่เป็นภาพก่อนนำไปสู่นามธรรม เพื่อช่วยให้ผู้เรียนสามารถเข้าใจเรื่องเศษส่วนในรูปแบบที่เป็นภาพได้อย่างชัดเจนก่อนที่จะเข้าใจในรูปแบบของสัญลักษณ์หรือสูตรคณิตศาสตร์

\subsection {ทฤษฎีที่เกี่ยวข้องกับการพัฒนาระบบ (System development theory)}
\subsubsection {Software development life cycle (SDLC)}
    โดย SDLC~\cite{sdlc} ที่เลือกใช้ในโครงงานนี้คือแบบ Agile model ซึ่งเป็นกระบวนการพัฒนาซอฟต์แวร์แบบทำไปปรับไป โดยจะมีการปรับปรุงและพัฒนาระบบอย่างต่อเนื่องตามความต้องการของผู้ใช้ โดยในแต่ละรอบของการพัฒนาจะมีการออกแบบ การพัฒนา การทดสอบ การส่งมอบระบบ และทบทวนสิ่งที่ทำไปในรอบนั้นๆเพื่อปรับปรุงในรอบถัดไป
    \\\hspace*{2em}เนื่องจากโครงงานนี้มีความต้องการที่อาจเปลี่ยนแปลงได้ตลอดเวลา และต้องการการตอบสนองที่รวดเร็วต่อความต้องการของผู้ใช้ Agile model จึงเป็นตัวเลือกที่เหมาะสม
    ซึ่งในขั้นตอนแต่ละขั้นตอนของ Agile model ที่คณะผู้จัดทำใช้นั้นมีรายละเอียดดังนี้
    \begin{itemize}
        \item Requirements: วางแผนการพัฒนา วิเคราะห์และกำหนดความต้องการของระบบ
        \item Design: ออกแบบระบบตามความต้องการที่ได้จากขั้นตอน requirements
        \item Development: พัฒนาระบบตามแบบที่ได้ออกแบบไว้
        \item Testing: ทดสอบระบบเพื่อหาข้อผิดพลาดและปรับปรุงระบบ
        \item Deployment: การให้ผู้ใช้ทดลองใช้งานระบบ
        \item Review: ทบทวนสิ่งที่ทำไปในรอบนั้น~ๆ~ เพื่อปรับปรุงในรอบถัดไป
    \end{itemize}
