\section{การค้นคว้าข้อมูล}
ในระยะเริ่มต้นของการดำเนินโครงงาน คณะผู้จัดทำได้มุ่งเน้นการสืบค้นและรวบรวมข้อมูลที่เกี่ยวข้อง เพื่อสร้างความเข้าใจอย่างถ่องแท้เกี่ยวกับปัญหาที่พบในการจัดการเรียนการสอนวิชาคณิตศาสตร์เรื่องเศษส่วน ตลอดจนศึกษาแนวทางการแก้ไขปัญหาที่มีอยู่ในปัจจุบัน

\subsection{วิเคราะห์ปัญหา}
คณะผู้จัดทำได้ตั้งข้อสังเกตเกี่ยวกับอุปสรรคในการเรียนรู้วิชาคณิตศาสตร์เรื่องเศษส่วนของนักเรียนระดับชั้นประถมศึกษาในปัจจุบัน จากการติดต่อสัมภาษณ์นักวิชาการจากสถาบันส่งเสริมการสอนวิทยาศาสตร์และเทคโนโลยี (สสวท.) พบว่า ปัญหาดังกล่าวเป็นสิ่งที่พบได้ทั่วไปในนักเรียนระดับประถมศึกษาทั่วโลก สาเหตุหลักเกิดจากเนื้อหาเรื่องเศษส่วนมีความเป็นนามธรรมสูงและมีรูปแบบที่แตกต่างไปจากจำนวนเต็มที่นักเรียนเคยศึกษามา ทำให้หลักการบางอย่างดูขัดแย้งกับความรู้เดิม จึงเป็นเรื่องยากที่จะทำให้ผู้เรียนทุกคนเกิดความเข้าใจที่ถูกต้องพร้อมกันผ่านวิธีการสอนเพียงรูปแบบเดียว

\subsection{วิเคราะห์วิธีแก้ปัญหาที่มีอยู่ในปัจจุบัน}
จากการวิเคราะห์ปัญหาเบื้องต้น คณะผู้จัดทำได้ศึกษาและรวบรวมวิธีการแก้ปัญหาที่ใช้กันอยู่ในปัจจุบัน เพื่อประเมินประสิทธิภาพและข้อจำกัดของแต่ละวิธี โดยมีรายละเอียดดังนี้
\begin{itemize}
    \item การเรียนรู้จากหนังสือเรียน - พบว่าหนังสือเรียนส่วนใหญ่มักใช้แบบจำลองพื้นที่ (Area Model) เช่น รูปภาพเค้ก พิซซ่า หรือตารางสี่เหลี่ยม ในการอธิบายเศษส่วน ส่งผลให้นักเรียนเกิดความยึดติดกับภาพจำเหล่านั้น และประสบปัญหาเมื่อต้องตีความเศษส่วนในรูปแบบอื่น นอกจากนี้ วิธีการดังกล่าวยังเหมาะสมกับการอธิบายเศษส่วนที่มีค่าน้อยกว่า 1 แต่อาจทำให้เกิดความสับสนเมื่อนำไปประยุกต์ใช้กับเนื้อหาที่ซับซ้อนขึ้น เช่น การบวกลบเศษส่วนที่มีค่ามาก หรือเศษเกิน
    \item การใช้สื่อการสอนจากสิ่งรอบตัว - เป็นแนวทางที่เน้นให้ผู้เรียนได้ลงมือปฏิบัติจริงและใช้จินตนาการในการแก้ปัญหา เช่น การพับกระดาษแบ่งเป็นส่วน ๆ หรือการเปรียบเทียบความยาวของไม้บรรทัดกับดินสอ วิธีนี้ช่วยลดความยึดติดกับภาพจำและเสริมสร้างความเข้าใจในหลักการของเศษส่วนได้ดี อย่างไรก็ตาม วิธีนี้มีข้อจำกัดด้านความสะดวก เนื่องจากต้องอาศัยเวลาและขั้นตอนในการเตรียมอุปกรณ์ที่ค่อนข้างมาก
    \item การใช้ซอฟต์แวร์ (Software) - เป็นวิธีการที่สามารถใช้เป็นสื่อการเรียนรู้ได้ทัดเทียมกับวิธีอื่น โดยอาศัยเพียงอุปกรณ์อิเล็กทรอนิกส์ เช่น คอมพิวเตอร์พกพา สมาร์ตโฟน หรือแท็บเล็ต อย่างไรก็ตาม ซอฟต์แวร์เพื่อการเรียนรู้เรื่องเศษส่วนในปัจจุบันยังไม่เป็นที่แพร่หลายนัก และส่วนใหญ่มักรองรับเพียงภาษาต่างประเทศ ทำให้กลายเป็นอุปสรรคในการเข้าถึงสำหรับนักเรียน
\end{itemize}
 
\subsection{สรุปปัญหาเบื้องต้น}
จากผลการวิเคราะห์ปัญหาและแนวทางการแก้ไขในปัจจุบัน สามารถสรุปได้ว่า การสร้างความเข้าใจและทำให้ผู้เรียนมองเห็นภาพที่ถูกต้องของเศษส่วนยังคงเป็นความท้าทายหลัก นอกจากนี้ วิธีการแก้ปัญหาที่มีอยู่บางส่วนยังมีข้อจำกัดด้านการเข้าถึงและความสะดวกในการใช้งาน ทำให้ยังไม่สามารถแก้ไขปัญหาได้อย่างมีประสิทธิภาพเท่าที่ควร

\subsection{เลือกพื้นที่สำรวจ}
คณะผู้จัดทำได้คัดเลือกโรงเรียนเป้าหมายที่เปิดสอนระดับชั้นประถมศึกษาตอนปลายในเขตพื้นที่จังหวัดเชียงใหม่ จำนวน 6 แห่ง ได้แก่ โรงเรียนพิงครัตน์ โรงเรียนพุทธิโศภน โรงเรียนดาราวิทยาลัย โรงเรียนบ้านเชิงดอยสุเทพ โรงเรียนโกวิทธำรงเชียงใหม่ และโรงเรียนสาธิตมหาวิทยาลัยเชียงใหม่ โดยเกณฑ์ในการคัดเลือกพิจารณาจากระยะทางที่สะดวกต่อการเดินทางจากมหาวิทยาลัยเชียงใหม่ และความเป็นไปได้ในการติดต่อขออนุญาตลงพื้นที่เก็บข้อมูลจากทางสถานศึกษา

\subsection{วางแผนสัมภาษณ์}
\begin{itemize}
    \item นักเรียน - มุ่งเน้นการสอบถามเกี่ยวกับกระบวนการคิดเพื่อประเมินระดับความเข้าใจเรื่องเศษส่วน สอบถามความรู้สึกและทัศนคติที่มีต่อวิชาคณิตศาสตร์ รวมถึงสำรวจความชื่นชอบในการใช้งานอุปกรณ์อิเล็กทรอนิกส์ เพื่อนำข้อมูลมาใช้ประกอบการตัดสินใจเลือกแพลตฟอร์มหลักสำหรับการพัฒนาโครงงาน
    \item ครูผู้สอน - มุ่งเน้นการสอบถามเกี่ยวกับวิธีการและเทคนิคการจัดการเรียนการสอน เพื่อรวบรวมข้อมูลเกี่ยวกับรูปแบบการสอนที่มีประสิทธิภาพจากประสบการณ์ตรงของผู้สอน รวมถึงสำรวจประเภทของสื่อและอุปกรณ์ที่ใช้ในชั้นเรียน เพื่อนำมาเป็นข้อมูลอ้างอิงในการออกแบบระบบ
\end{itemize}

\subsection{วางแผนออกสำรวจ}
การลงพื้นที่ในแต่ละสถานศึกษา มีวัตถุประสงค์เพื่อเก็บรวบรวมข้อมูลเชิงลึกเกี่ยวกับปัญหาที่เกิดขึ้นจริงในการเรียนการสอนวิชาคณิตศาสตร์เรื่องเศษส่วน โดยคณะผู้จัดทำได้กำหนดแผนการดำเนินการ ดังนี้
\begin{itemize}
    \item สัมภาษณ์นักเรียน จำนวน 6 คน ต่อโรงเรียน เนื่องจากข้อจำกัดด้านเวลาในการขออนุญาตเข้าสัมภาษณ์ในช่วงเวลาเรียน คณะผู้จัดทำจึงประเมินว่าจำนวนกลุ่มตัวอย่างดังกล่าวมีความเหมาะสมและสามารถดำเนินการให้แล้วเสร็จได้ภายในระยะเวลา 2 ชั่วโมง
    \item สัมภาษณ์ครูผู้สอน จำนวน 1 คน ต่อโรงเรียน เนื่องจากสถานศึกษาบางแห่งมีขนาดเล็กและมีครูผู้รับผิดชอบสอนวิชาคณิตศาสตร์ประจำระดับชั้นเพียงท่านเดียว
\end{itemize}

\subsection{เลือกพื้นที่สำรวจ}
คณะผู้จัดทำได้คัดเลือกโรงเรียนเป้าหมายที่เปิดสอนระดับชั้นประถมศึกษาตอนปลายในเขตพื้นที่จังหวัดเชียงใหม่ จำนวน 5 แห่ง 
ได้แก่ โรงเรียนพุทธิโศภน โรงเรียนดาราวิทยาลัย โรงเรียนบ้านเชิงดอยสุเทพ โรงเรียนโกวิทธำรงเชียงใหม่ และโรงเรียนสาธิตมหาวิทยาลัยเชียงใหม่ โดยเกณฑ์ในการคัดเลือกพิจารณาจากระยะทางที่สะดวกต่อการเดินทางจากมหาวิทยาลัยเชียงใหม่ 
และความเป็นไปได้ในการติดต่อขออนุญาตลงพื้นที่เก็บข้อมูลจากทางสถานศึกษา

\subsection{วางแผนสัมภาษณ์}
การสัมภาษณ์นี้จะสัมภาษณ์เพียงกับกลุ่มนักเรียนเท่านั้น โดยมุ่งเน้นการสอบถามเกี่ยวกับความรู้สึกและประสบการณ์ที่ได้จากการใช้งาน web application 
รวมถึงการสำรวจคำแนะนำที่สามารถนำมาใช้ในการพัฒนา web application นี้

\subsection{วางแผนออกสำรวจ}
การลงพื้นที่ในแต่ละสถานศึกษา มีวัตถุประสงค์เพื่อเก็บรวบรวมข้อมูลเชิงลึกเกี่ยวกับประสบการณ์การใช้งานและผลลัพธ์ที่ลัพธ์ที่ได้จากการใช้งาน web application โดยคณะผู้จัดทำได้กำหนดแผนการดำเนินการ ดังนี้
\begin{itemize}
    \item สัมภาษณ์นักเรียน จำนวน 6 คน ต่อโรงเรียน เนื่องจากข้อจำกัดด้านเวลาในการขออนุญาตเข้าสัมภาษณ์ในช่วงเวลาเรียน คณะผู้จัดทำจึงประเมินว่าจำนวนกลุ่มตัวอย่างดังกล่าวมีความเหมาะสมและสามารถดำเนินการให้แล้วเสร็จได้ภายในระยะเวลา 2 ชั่วโมง
\end{itemize}

\section{แนวคิดการออกแบบเนื้อหาการเรียนรู้}
เนื้อหาการเรียนรู้ใน web application นี้ถูกออกแบบโดยอิงจากหลักสูตรคณิตศาสตร์ของ สสวท. สำหรับระดับชั้นประถมศึกษา และการสัมภาษณ์ครูผู้สอนที่ได้ลงพื้นที่สำรวจในวิชา 269491
โดยมีการผสมผสานและจัดลำดับเนื้อหาให้มีการเชื่อมโยงกันระหว่างการวัดและเศษส่วน เพื่อนำเสนอเนื้อหาเศษส่วนออกมาในรูปแบบ Length model

\section{แนวคิดและหลักการออกแบบ User Interface และ User Experience}

\section{User Flow}
User flow ของ web application นี้ถูกออกแบบมาเพื่อให้ผู้ใช้งานสามารถเรียนรู้เรื่องเศษส่วนได้อย่างมีประสิทธิภาพและสนุกสนาน โดยมีการจัดลำดับเนื้อหาดังนี้
\begin{itemize}
    \item Login - เป็นหน้าที่ผู้ใช้จะเห็นเมื่อเข้าสู่ web application โดยหากผู้ใช้ยังไม่มีบัญชี สามารถสร้างบัญชีใหม่ได้ด้วยการกรอกชื่อผู้ใช้ใหม่ หากมีบัญชีอยู่แล้วสามารถเข้าสู่ระบบได้โดยการกดรูปโปรไฟล์และเลือกบัญชีที่ต้องการใช้งาน
    \item Map - เป็นหน้าที่แสดงแผนที่ของบทเรียนต่างๆ โดยบทเรียนที่ผู้ใช้สามารถเข้าเรียนได้จะแสดงเป็นปุ่มสีเข้ม ส่วนบทเรียนที่ยังไม่สามารถเข้าเรียนได้จะแสดงเป็นปุ่มสีอ่อน
    \item Episode - เป็นหน้าที่แสดงเนื้อหาของบทเรียน โดยในระหว่างการเรียนรู้จะมีวัตถุที่ผู้ใช้สามารถทำการโต้ตอบกับ web application ได้ เช่น การลากวาง การเลือกคำตอบ และการพิมพ์คำตอบ และเมื่อผู้ใช้เรียนรู้บทเรียนหนึ่งๆจบ
    จะมีการแสดงปุ่มให้ผู้ใช้เลือกว่าจะเรียนบทเรียนถัดไปหรือหยุดพักก่อน หากผู้ใช้เลือกที่จะเรียนบทเรียนถัดไป จะมีการนำผู้ใช้ไปยังหน้าของบทเรียนถัดไปทันที แต่หากผู้ใช้เลือกที่จะหยุดพักก่อน จะมีการนำผู้ใช้กลับไปยังหน้า Map เพื่อให้ผู้ใช้สามารถเลือกบทเรียนที่ต้องการเรียนได้ตามต้องการ
\end{itemize}

\subsection{การเรียนรู้}
การเรียนรู้เป็นฟีเจอร์ที่ทำให้ผู้ใช้งานสามารถเรียนรู้เนื้อหาที่คณะผู้จัดทำได้เตรียมไว้ การเรียนรู้จะดำเนินผ่านหน้า web 
โดยหน้า web นี้จะมีส่วนกล่องข้อความและมาสคอตที่คอยบอกเล่าเนื้อหาของบทเรียน และในระหว่างการเรียนรู้จะมีวัตถุที่ผู้ใช้สามารถทำการโต้ตอบกับ web application ได้ 
โดยจะมีคำสั่งว่าผู้ใช้ต้องตอบโต้อย่างไรแสดงในกล่องข้อความ และรูปแบบของการโต้ตอบมีดังนี้
 \begin{itemize}
    \item การลากวาง - เป็นการโต้ตอบที่มีวัตถุประสงค์ให้ผู้ใช้ลากวัตถุที่แสดงบนหน้า web ไปยังจุดรับวัตถุที่กำหนดไว้ 
    ผู้ใช้ต้องลากวัตถุที่ถูกต้องไปยังจุดรับวัตถุเพื่อดำเนินเนื้อหาต่อไป หากผู้ใช้ลากวัตถุที่ไม่ถูกต้องไปยังจุดรับวัตถุจะไม่มีสิ่งใดเกิดขึ้น 
    เพียงแต่เนื้อหาจะไม่ดำเนินต่อจนกว่าผู้ใช้จะลากวัตถุที่ถูกต้องไปยังจุดรับวัตถุ
    \item การเลือกคำตอบ - เป็นการโต้ตอบที่มีวัตถุประสงค์ให้ผู้ใช้เลือกคำตอบที่แสดงบนหน้า web 
    ผู้ใช้ต้องเลือกคำตอบที่ถูกต้องเพื่อดำเนินเนื้อหาต่อไป หากผู้ใช้เลือกคำตอบที่ผิดจะมีการอธิบายว่าเพราะเหตุใดคำตอบนี้ถึงไม่ถูกต้องและเนื้อหาจะดำเนินต่อไป
    \item การพิมพ์คำตอบ - เป็นการโต้ตอบที่มีวัตถุประสงค์ให้ผู้ใช้พิมพ์คำตอบลงในกล่องคำตอบที่แสดงบนหน้า web 
    ผู้ใช้ต้องพิมพ์คำตอบที่ถูกต้องลงในกล่องคำตอบเพื่อดำเนินเนื้อหาต่อไป หากผู้ใช้พิมพ์คำตอบที่ไม่ถูกต้องลงในกล่องคำตอบ ระบบจะแสดงข้อความว่าคำตอบไม่ถูกต้องและให้ผู้ใช้แก้คำตอบ
    \item การเลือกหลังจบบทเรียน - เป็นการโต้ตอบจะแสดงขึ้นบนหน้า web เมื่อผู้ใช้เรียนรู้บนเรียนหนึ่งๆจบ 
    โดยมีวัตถุประสงค์ให้ผู้ใช้เลือกว่าต้องการที่จะเรียนรู้เนื้อหาถัดไปหรือหยุดไว้ก่อนแล้วพักผ่อน
 \end{itemize}

\section{ขั้นตอนการทำงานของระบบ}
web application นี้เริ่มต้นทำงานเมื่อผู้ใช้เปิดใช้งาน web application ผ่าน web browser จากนั้นระบบจะให้ผู้ใช้ทำการ login เพื่อเข้าสู่การเรียนรู้เนื้อหาที่คณะผู้จัดทำได้เตรียนไว้ 
โดยการ login สามารถทำได้ 2 แบบดังนี้
\begin{enumerate}
    \item ไม่มีบัญชีผู้ใช้ - ผู้ใช้เข้าใช้งาน web application นี้เป็นครั้งแรก ระบบจะให้ผู้ใช้สร้างบัญชีผู้ใช้ใหม่และเข้าสู่บนนำเพื่อเรียนรู้เกี่ยวกับพื้นฐานการใช้งาน web application นี้
    \item มีบัญชีผู้ใช้ - ระบบจะแสดงบัญชีที่เคยถูกสร้างไว้ ผู้ใช้สามารถ login ด้วยบัญชีนั้นและเข้าสู่การเรียนรู้ได้ทันที
\end{enumerate}
\hspace*{2em}เมื่อผู้ใช้ทำการ login เรียบร้อยแล้วผู้ใช้จะเข้าสู่หน้า map ที่แสดงความคืบหน้าในการเรียนรู้ของผู้ใช้และแสดงบทเรียนทั้งหมดที่มีอยู่ในระบบ โดยบทเรียนจะถูกแบ่งเป็น episode 
ผู้ใช้สามารถเลือกเนื้อหาที่ต้องการเรียนรู้ได้ด้วยตนเอง หลังจากเลือกเนื้อหาเรียบร้อยแล้วผู้ใช้จะเข้าสู่การเรียนรู้บทเรียนที่ถูกเลือก ระบบจะทำการแสดงเนื้อหาให้ผู้ใช้อ่านหรือตอบโต้เพื่อเรียนรู้เนื้อหาบนหน้า web 
โดยระบบจะตรวจสอบการตอบโต้ของผู้ใช้ ซึ่งการตอบโต้ที่ถูกต้องของผู้ใช้จะทำให้เนื้อหาการเรียนรู้ดำเนินต่อไปได้ การตอบโต้ที่ไม่ถูกต้องจะทำให้เนื้อหาไม่ดำเนินต่อแต่ระบบจะให้โอกาสผู้ใช้ได้แก้การตอบโต้นั้นจนกว่าผู้ใช้จะทำการตอบโต้ที่ถูกต้อง 
\hspace*{2em}เมื่อผู้ใช้เรียนรู้เนื้อหาของบทเรียนหนึ่งจบแล้ว ระบบจะบันทึกความคืบหน้าในการเรียนรู้ของผู้ใช้และแสดงตัวเลือกหลังจบบทเรียนที่ผู้ใช้สามารถเลือกได้ว่า ผู้ใช้ต้องการที่จะไปที่หน้า map เพื่อหยุดพัก 
หรือเข้าสู่บทเรียนต่อไป
