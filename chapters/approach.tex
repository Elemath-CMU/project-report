\section{การค้นคว้าข้อมูล}
ในระยะเริ่มต้นของการดำเนินโครงงาน คณะผู้จัดทำได้มุ่งเน้นการสืบค้นและรวบรวมข้อมูลที่เกี่ยวข้อง 
เพื่อสร้างความเข้าใจอย่างถ่องแท้เกี่ยวกับปัญหาที่พบในการจัดการเรียนการสอนวิชาคณิตศาสตร์เรื่องเศษส่วน ตลอดจนศึกษาแนวทางการแก้ไขปัญหาที่มีอยู่ในปัจจุบัน

\subsection{ปัญหาที่มีอยู่ในปัจจุบัน}
คณะผู้จัดทำเกิดข้อสงสัยว่าสิ่งใดเป็นอุปสรรคในการเรียนรู้วิชาคณิตศาสตร์เรื่องเศษส่วนของนักเรียนไทยระดับชั้นประถมศึกษาในปัจจุบัน 
จากการติดต่อสัมภาษณ์นักวิชาการจากสถาบันส่งเสริมการสอนวิทยาศาสตร์และเทคโนโลยี (สสวท.) ได้ข้อมูลว่า 
อุปสรรคในการเรียนรู้วิชาคณิตศาสตร์เรื่องเศษส่วนของนักเรียนระดับประถมศึกษาทั่วโลก เกิดขึ้นมาจากเนื้อหาเรื่องเศษส่วนมีความเป็นนามธรรมสูงและมีรูปแบบที่แตกต่างไปจากจำนวนเต็มที่นักเรียนเคยศึกษามา 
ทำให้หลักการบางอย่างดูขัดแย้งกับความรู้เดิม และเรื่องยากที่จะทำให้ผู้เรียนทุกคนเกิดความเข้าใจที่ถูกต้องพร้อมกันผ่านวิธีการสอนเพียงรูปแบบเดียว

\subsection{วิธีแก้ปัญหาที่มีอยู่ในปัจจุบัน}
จากปัญหาข้างต้น คณะผู้จัดทำได้รวบรวมและศึกษาวิธีการแก้ปัญหาที่ใช้กันอยู่ในการศึกษาไทยในปัจจุบัน เพื่อประเมินประสิทธิภาพและข้อจำกัดของแต่ละวิธี โดยมีรายละเอียดดังนี้
\begin{itemize}
    \item การเรียนรู้จากหนังสือเรียน พบว่าหนังสือเรียนส่วนใหญ่มักใช้แบบจำลองพื้นที่ (area model) เช่น รูปภาพเค้ก พิซซ่า หรือตารางสี่เหลี่ยม ในการอธิบายเศษส่วน 
    วิธีนี้สามารถลดปัญหาความเป็นนามธรรมของเศษส่วนได้แต่ส่งผลให้นักเรียนเกิดความยึดติดกับภาพจำเหล่านั้น และประสบปัญหาเมื่อต้องตีความเศษส่วนในรูปแบบอื่น นอกจากนี้ 
    วิธีการดังกล่าวยังเหมาะสมกับการอธิบายเศษส่วนที่มีค่าน้อยกว่า 1 แต่อาจทำให้เกิดความสับสนเมื่อนำไปประยุกต์ใช้กับเนื้อหาที่ซับซ้อนขึ้น 
    เช่น การบวกลบเศษส่วนที่มีค่ามาก เศษเกิน และจำนวนคละ
    \item การใช้สื่อการสอนจากสิ่งรอบตัว เป็นแนวทางที่เน้นให้ผู้เรียนได้ลงมือปฏิบัติจริงและใช้จินตนาการในการแก้ปัญหา 
    เช่น การพับกระดาษแบ่งเป็นส่วน ๆ หรือการเปรียบเทียบความยาวของไม้บรรทัดกับดินสอ 
    วิธีนี้ช่วยลดความยึดติดกับภาพจำและเสริมสร้างความเข้าใจในหลักการของเศษส่วนได้ดี อย่างไรก็ตาม วิธีนี้มีข้อจำกัดด้านความสะดวก 
    เนื่องจากต้องอาศัยเวลาและขั้นตอนในการเตรียมอุปกรณ์ที่ค่อนข้างมาก
    \item การใช้ซอฟต์แวร์ (software) เป็นวิธีการที่สามารถใช้เป็นสื่อการเรียนรู้ได้ทัดเทียมกับวิธีอื่น โดยอาศัยเพียงอุปกรณ์อิเล็กทรอนิกส์ 
    เช่น คอมพิวเตอร์พกพา สมาร์ตโฟน หรือแท็บเล็ต อย่างไรก็ตาม ซอฟต์แวร์เพื่อการเรียนรู้เรื่องเศษส่วนในปัจจุบันยังไม่เป็นที่แพร่หลายนัก 
    และส่วนใหญ่มักรองรับเพียงภาษาต่างประเทศ ทำให้กลายเป็นอุปสรรคในการเข้าถึงสำหรับนักเรียน
\end{itemize}
 
\subsection{ปัญหาที่เหลืออยู่}
จากผลการวิเคราะห์ปัญหาและแนวทางการแก้ไขในการศึกษาไทยในปัจจุบัน ทำให้เราเห็นว่า การสร้างความเข้าใจและทำให้ผู้เรียนมองเห็นภาพที่ถูกต้องของเศษส่วนยังคงเป็นความท้าทายหลัก นอกจากนี้ วิธีการแก้ปัญหาที่มีอยู่บางส่วนยังมีข้อจำกัดด้านการเข้าถึงและความสะดวกในการใช้งาน ทำให้ยังไม่สามารถแก้ไขปัญหาได้อย่างมีประสิทธิภาพเท่าที่ควร

\section{Survey 1: Problem survey}
จากปัญหาที่เกิดขึ้นจริงในการเรียนการสอนวิชาคณิตศาสตร์เรื่องเศษส่วน คณะผู้จัดทำจึงได้ทำการลงพื้นครั้งที่ 1 มีจุดประสงค์เพื่อยืนยันปัญหา
และเก็บรวบรวมข้อมูลเชิงลึกเกี่ยวกับการเรียนรู้และกระบวนการเรียนการสอนวิชาคณิตศาสตร์เรื่องเศษส่วนที่เกิดขึ้นจริง โดยในแต่ละพื้นที่จะดำเนินการสัมภาษณ์กับบุคคลคน 2 กลุ่ม ได้แก่
\begin{itemize}
    \item นักเรียน จำนวน 6 คน: มุ่งเน้นการสอบถามเกี่ยวกับกระบวนการคิดเพื่อประเมินระดับความเข้าใจเรื่องเศษส่วน สอบถามความรู้สึกและทัศนคติที่มีต่อวิชาคณิตศาสตร์ รวมถึงสำรวจความชื่นชอบในการใช้งานอุปกรณ์อิเล็กทรอนิกส์ เพื่อนำข้อมูลมาใช้ประกอบการตัดสินใจเลือกแพลตฟอร์มหลักสำหรับการพัฒนาโครงงาน
    \item ครูผู้สอน จำนวน 1 คน: มุ่งเน้นการสอบถามเกี่ยวกับวิธีการและเทคนิคการจัดการเรียนการสอน เพื่อรวบรวมข้อมูลเกี่ยวกับรูปแบบการสอนที่มีประสิทธิภาพจากประสบการณ์ตรงของผู้สอน รวมถึงสำรวจประเภทของสื่อและอุปกรณ์ที่ใช้ในชั้นเรียน เพื่อนำมาเป็นข้อมูลอ้างอิงในการออกแบบระบบ
\end{itemize}

ในส่วนของพื้นที่ คณะผู้จัดทำได้คัดเลือกโรงเรียนเป้าหมายที่เปิดสอนระดับชั้นประถมศึกษาตอนปลายในเขตพื้นที่จังหวัดเชียงใหม่ จำนวน 6 แห่ง 
ได้แก่ โรงเรียนพิงครัตน์ โรงเรียนพุทธิโสภน โรงเรียนดาราวิทยาลัย โรงเรียนบ้านเชิงดอยสุเทพ โรงเรียนโกวิทธำรงเชียงใหม่ และโรงเรียนสาธิตมหาวิทยาลัยเชียงใหม่ 
โดยเกณฑ์ในการคัดเลือกพิจารณาจากระยะทางที่สะดวกต่อการเดินทางจากมหาวิทยาลัยเชียงใหม่ และความเป็นไปได้ในการติดต่อขออนุญาตลงพื้นที่เก็บข้อมูลจากทางสถานศึกษา

\section{แนวคิดการออกแบบเนื้อหาการเรียนรู้}
เนื้อหาการเรียนรู้ใน web application นี้ถูกออกแบบโดยอิงจากหลักสูตรคณิตศาสตร์ของ สสวท. สำหรับระดับชั้นประถมศึกษา และการสัมภาษณ์ครูผู้สอนที่ได้ลงพื้นที่สำรวจในวิชา 261491
โดยมีการผสมผสานและจัดลำดับเนื้อหาให้มีการเชื่อมโยงกันระหว่างการวัดและเศษส่วน เพื่อนำเสนอเนื้อหาเศษส่วนออกมาในรูปแบบ length model

\section{User flow}
User flow ของ web application นี้ถูกออกแบบมาเพื่อให้ผู้ใช้งานสามารถเรียนรู้เรื่องเศษส่วนได้อย่างมีประสิทธิภาพและสนุกสนาน โดยมีการจัดลำดับเนื้อหาดังนี้
\begin{itemize}
    \item การสร้างบัญชีผู้ใช้ใหม่ หรือการเข้าสู่ระบบด้วยบัญชีที่มีอยู่แล้ว เพื่อให้ผู้ใช้งานสามารถบันทึกความคืบหน้าในการเรียนรู้ได้
    \item การเลือกบทเรียนที่ต้องการเรียนรู้ โดยจำนวนบทเรียนที่เข้าถึงได้จะเพิ่มขึ้นตามความคืบหน้าของผู้ใช้งาน
    \item การเรียนรู้ในแต่ละบทเรียน โดยแต่ละบทเรียนจะมีเนื้อหาและกิจกรรมที่แตกต่างกันออกไป โดยหลังจากจบบทเรียน ผู้ใช้งานสามารถเลือกที่จะเรียนเนื้อหาถัดไป หรือหยุดพักก่อนแล้วกลับมาเรียนต่อในภายหลังได้
\end{itemize}

\subsection{การสร้างบัญชี}
การสร้างบัญชีเป็นฟีเจอร์ที่ทำให้ผู้ใช้สามารถสร้างบัญชีสำหรับใช้งาน web application โดยบัญชีนี้จะบันทึกข้อมูลความคืบหน้าการเรียนรู้ของผู้ใช้ 
ทำให้ผู้ใช้สามารถหยุดพักได้ทุกเมื่อและสามารถกลับเข้ามาเรียนรู้ต่อจากครั้งก่อนได้

และรูปแบบของการมีปฏิสัมพันธ์เป็นดังนี้
 \begin{itemize}
    \item การลากวาง เป็นการมีปฏิสัมพันธ์ที่มีวัตถุประสงค์ให้ผู้ใช้ลากวัตถุที่แสดงบนหน้า web ไปยังจุดรับวัตถุที่กำหนดไว้ 
    ผู้ใช้ต้องลากวัตถุที่ถูกต้องไปยังจุดรับวัตถุเพื่อดำเนินเนื้อหาต่อไป หากผู้ใช้ลากวัตถุที่ไม่ถูกต้องไปยังจุดรับวัตถุจะไม่มีสิ่งใดเกิดขึ้น 
    เพียงแต่เนื้อหาจะไม่ดำเนินต่อจนกว่าผู้ใช้จะลากวัตถุที่ถูกต้องไปยังจุดรับวัตถุ

\section{แนวคิดการออกแบบโครงสร้างข้อมูลเกม (Game Configuration Design)}
ในการพัฒนาแอปพลิเคชันเกมเพื่อการเรียนรู้เรื่องเศษส่วน ระบบได้ถูกออกแบบให้สามารถกำหนดพฤติกรรมของเกมผ่านโครงสร้างข้อมูล (Configuration-based Design) 
เพื่อให้สามารถปรับเปลี่ยนเนื้อหาและลำดับการเล่นได้อย่างยืดหยุ่น โดยไม่จำเป็นต้องแก้ไขโค้ดหลักของระบบ

โครงสร้างของเกมถูกแบ่งออกเป็นหน่วยย่อยเรียกว่า “Episode” ซึ่งแต่ละ Episode ประกอบด้วยลำดับของเหตุการณ์ (Interaction) 
ที่ผู้เล่นต้องดำเนินการตามลำดับ โดยข้อมูลทั้งหมดจะถูกจัดเก็บในรูปแบบของอาเรย์ gameData ซึ่งรวม Episode ทั้งหมดเข้าด้วยกัน

\subsection{โครงสร้างของ Episode และ Interaction}
ในแต่ละ Episode จะประกอบไปด้วย Interaction หลายรายการ โดยแต่ละ Interaction จะมีประเภทที่แตกต่างกัน เพื่อรองรับรูปแบบการเรียนรู้ที่หลากหลาย ได้แก่ 
\begin{itemize}
    \item playground: เป็นฉากที่ผู้เล่นสามารถโต้ตอบกับวัตถุ เช่น การลากวาง หรือการป้อนคำตอบ
    \item choiceAnswer: เป็นรูปแบบคำถามแบบเลือกคำตอบ โดยผู้เล่นเลือกตัวเลือกเพื่อดำเนินเนื้อเรื่อง
    \item checkpoint: เป็นจุดพักหรือข้อความเนื้อเรื่องที่ไม่มีการโต้ตอบ
\end{itemize}
โครงสร้างดังกล่าวช่วยให้สามารถออกแบบลำดับการเรียนรู้แบบเป็นขั้นตอน (step-by-step learning) และรองรับการแตกแขนงของเนื้อหาได้
\subsection{การออกแบบวัตถุภายในเกม (Object Model)}
ใน Interaction ประเภทที่มีการโต้ตอบ ระบบรองรับการแสดงวัตถุ (Object) ภายในฉาก โดยวัตถุแต่ละชนิดถูกออกแบบให้มีคุณสมบัติเฉพาะ เช่น
\begin{itemize}
    \item วัตถุประเภท pencil และ stick ใช้แทนหน่วยวัดความยาว
    \item วัตถุประเภท spawner ใช้สำหรับสร้างวัตถุใหม่ในระหว่างการเล่น
    \item วัตถุประเภท fractionInput ใช้สำหรับให้ผู้เล่นกรอกค่าเศษส่วน
    \item วัตถุประเภท other ใช้แสดงองค์ประกอบทั่วไป เช่น รูปภาพ
\end{itemize}
วัตถุทั้งหมดจะมีพิกัดตำแหน่ง (x, y) และสามารถกำหนดให้เคลื่อนที่ได้หรือไม่ผ่านคุณสมบัติ fixed 
การออกแบบในลักษณะนี้ช่วยให้สามารถจำลองสถานการณ์เชิงภาพ (visual interaction) เพื่อสนับสนุนการเรียนรู้แบบเป็นรูปธรรม (concrete learning)

\subsection{ระบบตรวจสอบคำตอบ (Rule System)}
ระบบใช้กลไก “Rule-based” ในการตรวจสอบการกระทำของผู้เล่น โดยแต่ละ Interaction สามารถกำหนดกฎ (Rule) เพื่อใช้ตรวจสอบความถูกต้องและกำหนดการเปลี่ยนไปยัง Interaction ถัดไป 
ประเภทของกฎที่รองรับ เช่น
\begin{itemize}
    \item การลากวัตถุไปวางบนวัตถุอื่น (dropOnObject)
    \item การวางวัตถุในพื้นที่ที่กำหนด (dropOnArea, dropInsideArea)
    \item การจัดตำแหน่งวัตถุให้ตรงตำแหน่ง (snapToPosition)
    \item การกรอกค่าเศษส่วน (enterFraction)
    \item การดำเนินต่อเมื่อบทสนทนาจบ (lastDialogue)
\end{itemize}
เมื่อผู้เล่นทำเงื่อนไขสำเร็จ ระบบจะเปลี่ยนไปยัง Interaction ถัดไปตามค่าที่กำหนด (nextInteraction) แนวทางนี้ช่วยให้สามารถออกแบบเงื่อนไขการเรียนรู้ที่หลากหลาย และควบคุมลำดับการเรียนรู้ได้อย่างแม่นยำ

\subsection{แนวทางการออกแบบเพื่อความยืดหยุ่น}
เพื่อให้ระบบสามารถขยายต่อได้ง่าย ได้มีการกำหนดแนวทางในการเขียนข้อมูล ดังนี้
\begin{enumerate}
    \item กำหนดหมายเลข Interaction ไม่ให้ซ้ำกันภายใน Episode
    \item กำหนดการเชื่อมโยง nextInteraction ให้ถูกต้อง
    \item ใช้ id ของวัตถุอย่างสม่ำเสมอเพื่อให้ Rule ทำงานได้ถูกต้อง
    \item แยก Interaction ออกเป็นขั้นตอนย่อยเพื่อลดความซับซ้อน
\end{enumerate}
แนวทางดังกล่าวช่วยให้การพัฒนาเนื้อหาใหม่สามารถทำได้ง่าย และลดความผิดพลาดในการออกแบบเกม

\section{เนื้อหาในบทเรียน}
เนื้อหาในบทเรียนถูกออกแบบมาเพื่อให้ผู้ใช้งานสามารถเรียนรู้เรื่องเศษส่วน ซึ่งเนื้อหาจะมีความเชื่อมโยงกันและค่อยๆเพิ่มความซับซ้อนขึ้นตามลำดับ
โดยเนื้อหาทั้งหมดในบทเรียนจะถูกแบ่งออกเป็น 9 episode ซึ่งแต่ละ episode จะมีเนื้อหาดังนี้

    เพื่อให้ผู้ใช้งานได้เห็นภาพของการแปลงค่าเศษส่วนมากยิ่งขึ้น สำหรับบทเรียนนี้ มีเป้าหมายให้ผู้ใช้งานเข้าใจถึงความสำคัญและวิธิการในการแปลงค่าเศษส่วน

    \subsection{Episode 9: การบวกเศษส่วนที่มีตัวส่วนไม่เท่ากัน}
    ผู้ใช้งานจะได้เรียนรู้เกี่ยวกับการบวกเศษส่วนที่มีตัวส่วนไม่เท่ากัน ซึ่งเนื้อหาในบทเรียนนี้ได้มีการใช้เนื้อหาและปฏิสัมพันธ์ที่มีการใช้ในบทก่อน~ๆ~มาประกอบเพื่อช่วยให้ผู้ใช้งานเข้าใจว่าการบวกเศษส่วนที่มีตัวส่วนไม่เท่ากันนั้นสามารถทำได้จากการแปลงค่าเศษส่วนให้มีตัวส่วนเท่ากันก่อนแล้วจึงค่อยบวกตัวเศษเข้าด้วยกัน
    โดยในบทเรียนนี้จะให้ผู้ใช้งานได้ทำการลากและวางแท่งไม้ลงบนเส้นจำนวน ก่อนที่มาสคอตให้ผู้ใช้งานทำการแปลงค่าเศษส่วนของแท่งไม้ด้วยการลากแท่งไม้ที่มีความยาวเท่ากันแต่ถูกแปลงค่าแล้วมาทับแท่งไม้เดิม แล้วจึงให้ผู้ใช้งานทำการบวกเศษส่วนที่อยู่บนเส้นจำนวนและตอบลงไปในกล่องคำตอบที่กำหนดไว้
    สำหรับบทเรียนนี้ มีเป้าหมายให้ผู้ใช้งานได้รู้จักกับวิธีการที่ใช้ในการบวกเศษส่วนที่มีตัวส่วนไม่เท่ากัน และสามารถอธิบายได้ว่าเพราะเหตุใดจึงต้องแปลงค่าเศษส่วนก่อนที่จะบวกตัวเศษเข้าด้วยกัน

\section{Survey 2: Usability testing}
ในช่วงที่คณะผู้จัดทำยังพัฒนา web application ไม่สำเร็จครบทั้ง 9 episode เป็นช่วงเดียวกันกับที่หลายโรงเรียนที่คณะผู้จัดทำเคยลงพื้นที่ในครั้งที่ 1 อยู่ในช่วงสอบปลายภาคและเตรียมปิดเทอม 
ทำให้การลงพื้นที่ครั้งที่ 2 นี้เป็นการลงพื้นที่เพื่อทดสอบและเก็บข้อมูลการทำงานฟีเจอร์ทั้งหมดที่มีอยู่ในระบบเพื่อนำไปปรับให้เหมาะสมกับการใช้งานของนักเรียน

การสัมภาษณ์ในการลงพื้นที่ครั้งที่ 2 นี้จะดำเนินกับนักเรียนจำนวน 6 คนจากโรงเรียน 3 แห่ง ได้แก่ โรงเรียนสาธิตมหาวิทยาลัยเชียงใหม่ โรงเรียนดาราวิทยาลัย และโรงเรียนโกวิทธํารงเชียงใหม่ 
อีกทั้งยังมีการสัมภาษณ์นักเรียนจำนวน 3 คนจากโรงเรียนพิงครัตน์ โดยการสัมภาษณ์จะมุ่งเน้นที่การสอบถามเกี่ยวกับความถูกต้องในการทำงานของฟีเจอร์ต่าง~ๆ~ 
สอบถามความรู้สึกที่มีต่อองค์ประกอบของ web application และสอบถามสิ่งที่นักเรียนต้องการให้ปรับแก้หรือเพิ่มเข้ามาใน web application นี้

\section{Survey 3: Usability testing และ evaluation}
การลงพื้นที่ครั้งที่ 3 นี้ เกิดขึ้นหลังจากที่คณะผู้จัดทำได้นำข้อมูลที่ได้จากการลงพื้นที่ครั้งที่ 2 มาพัฒนา web application สำเร็จครบ 9 episode ตามแผนที่เราวางไว้ 
ด้วยเหตุนี้คณะผู้จัดทำจึงดำเนินการลงพื้นที่ครั้งที่ 3 โดยมีวัตถุประสงค์เพื่อทดสอบการทำงานของ web application ทุก episode และ ประเมินผลสัมฤทธิ์ทางการเรียนจากการใช้งาน web application

การสัมภาษณ์ในการลงพื้นที่ครั้งที่ 2 นี้จะดำเนินกับนักเรียนจำนวน 6 คนจากโรงเรียน 2 แห่ง ได้แก่ โรงเรียนบ้านเชิงดอยสุเทพ และโรงเรียนพุทธิโสภน 
โดยการสัมภาษณ์จะมุ่งเน้นที่การสอบถามเกี่ยวกับความถูกต้องในการทำงานของฟีเจอร์ต่าง~ๆ~
