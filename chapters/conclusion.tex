\chapter{\ifenglish Conclusions and Discussions\else บทสรุปและข้อเสนอแนะ\fi}

\section{\ifenglish Conclusions\else สรุปผล\fi}
โครงงานนี้มีวัตถุประสงค์ในการพัฒนาระบบเพื่อช่วยสอนเรื่องเศษส่วนสำหรับนักเรียนระดับประถมศึกษา โดยมีการพัฒนา UX/UI ที่เป็นมิตรและการทำงานของระบบที่ยืดหยุ่น เพื่อให้ผู้เรียนสามารถเข้าใจและมีส่วนร่วมในการเรียนรู้เรื่องเศษส่วนได้อย่างมีประสิทธิภาพ
โดยจากการประเมินระบบผ่านการลงพื้นที่ทดสอบกับนักเรียนจริง คณะผู้จัดทำได้พบว่าระบบสามารถสนับสนุนการเรียนรู้เรื่องเศษส่วนได้อย่างมีประสิทธิภาพ
ในด้านของฟังก์ชันการใช้งาน พบว่าระบบสามารถรองรับการเรียนรู้ผ่านกิจกรรมต่าง~ๆ ได้ครบตามกรอบที่ตั้งไว้ โดยคณะผู้จัดทำได้ใช้แนวคิดในการใช้ component และการจัดการข้อมูลซ้ำมาใช้ เพื่อช่วยให้กระบวนการพัฒนาเสร็จภายในระยะเวลาที่กำหนด อีกทั้งยังลดงานซ้ำซ้อนและการเกิดบั๊กที่อาจเกิดขึ้นจากการเขียนโค้ดซ้ำ
ในด้านประสบการณ์ผู้ใช้และงานออกแบบ UX/UI พบว่าโครงงานนี้ ช่วยให้ผู้เรียนเห็นภาพและเข้าใจเรื่องเศษส่วนได้ชัดเจน พร้อมทั้งได้รับการตอบรับที่ดีจากนักเรียนในเรื่องของความน่าสนใจในการใช้งานระบบ ซึ่งเป็นปัจจัยสำคัญที่ช่วยกระตุ้นการมีส่วนร่วมและส่งเสริมการเรียนรู้ของผู้เรียนได้
จากบทสรุปเชิงปฏิบัติ คณะผู้จัดทำได้เรียนรู้เกี่ยวกับการจัดการในการทำโครงงานสองประการคือ 1) การวางแผนแบ่งงานและสื่อสารภายในกลุ่มอย่างสม่ำเสมอ และ 2) การปรับแก้ไทม์ไลน์เมื่อพบคอขวดในการออกแบบ โดยรักษาคุณภาพงานและเสร็จตามแผน
ทั้งหมดนี้สะท้อนให้เห็นว่าโครงงานประสบความสำเร็จในระดับที่น่าพอใจตามเป้าหมายหลัก พร้อมทั้งได้บทเรียนสำคัญสำหรับการพัฒนาและขยายต่อไปในอนาคต

\section{\ifenglish Challenges\else ปัญหาที่พบและแนวทางการแก้ไข\fi}
ในการทำโครงงานนี้ พบว่าเกิดปัญหาหลัก ๆ ดังนี้

\subsection*{ปัญหาที่พบ และได้รับการแก้ไขแล้ว}
\begin{enumerate}
    \item เกิดความล่าช้าในการออกแบบ UX/UI สาเหตุมาจากความไม่รู้ในสิ่งที่เราไม่รู้ว่าเราไม่รู้ เนื่องจากทางผู้จัดทำไม่ทราบถึงรายละเอียดเชิงลึกว่า การออกแบบสื่อการสอนในรูปแบบเกมสำหรับเด็กระดับประถมศึกษานั้น 
    มีความละเอียดอ่อนและซับซ้อนสูงมาก ประกอบกับต้องออกแบบเนื้อหาการเรียนรู้ควบคู่กันไปอย่างเป็นระบบ ส่งผลให้ใช้ระยะเวลามากกว่าที่วางแผนไว้ ทางผู้จัดทำจึงแก้ไขปัญหาโดยการปรับแผนการทำงานใหม่ 
    โดยแบ่งงานออกเป็นส่วนย่อย ๆ และกำหนดเวลาสำหรับแต่ละส่วนอย่างชัดเจน
    \item เกิดปัญหาคอขวด (bottleneck) ในกระบวนการทำงาน สาเหตุมาจากการจัดสรรทรัพยากรบุคคลไม่สอดคล้องกับปริมาณงานในช่วงเริ่มต้น โดยกำหนดให้มีผู้รับผิดชอบในส่วนของการออกแบบเพียงคนเดียว 
    ส่งผลให้ขั้นตอนการพัฒนาระบบ ที่ต้องรอการออกแบบ UI ไม่สามารถดำเนินการต่อได้ ทางผู้จัดทำจึงแก้ไขปัญหาโดยการปรับโครงสร้างการทำงาน ดึงสมาชิกเข้ามาช่วยรับผิดชอบในส่วนของการออกแบบเพิ่มเติม 
    เพื่อลดระยะเวลารอคอยของฝ่ายอื่น ๆ ทำให้กระบวนการทำงานในภาพรวมสามารถดำเนินต่อไปได้ตามกำหนดการ
    \item การกระจายภาระงานที่ไม่สมดุล สาเหตุมาจากการขาดการแจกแจงรายละเอียดโครงสร้างงาน (task breakdown) ที่ชัดเจนตั้งแต่เริ่มโปรเจกต์ ทำให้ในช่วงแรกของการดำเนินงาน 
    การแบ่งหน้าที่รับผิดชอบของสมาชิกในกลุ่มยังไม่สอดคล้องกับปริมาณงานที่เกิดขึ้นจริง ส่งผลให้สมาชิกบางคนมีภาระงานที่หนักเกินไป ทางผู้จัดทำจึงแก้ไขโดยการนำกระบวนการทำงานทั้งหมดมาทบทวนร่วมกันใหม่ 
    และจัดสรรภาระงานให้มีความสมดุลมากขึ้น เพื่อให้การดำเนินงานเป็นไปอย่างมีประสิทธิภาพและเสร็จสิ้นตามกำหนดเวลา
    \item การสูญหายของเอกสารหนังสือเชิญ ในขั้นตอนการติดต่อประสานงาน ทางโรงเรียนที่เป็นกลุ่มเป้าหมายได้ทำเอกสารหนังสือเชิญสูญหาย 
    ส่งผลให้การติดต่อเพื่อขอลงพื้นที่เกิดความล่าช้า ทางผู้จัดทำจึงแก้ไขโดยการติดต่อประสานงานกับเจ้าหน้าที่ธุรการของภาควิชาวิศวกรรมคอมพิวเตอร์ 
    เพื่อขอออกสำเนาหนังสือเชิญอีกครั้ง และนำไปส่งมอบให้แก่ทางโรงเรียนใหม่เพื่อดำเนินการในขั้นต่อไป
    \item เกิดปัญหา LaTeX compile เอกสารใน Windows ไม่ได้ สาเหตุมาจากการที่ระบบปฏิบัติการขาดชุดแบบอักษร GYRE ทางผู้จัดทำจึงแก้ไขโดยการดาวน์โหลดและติดตั้งชุดแบบอักษรดังกล่าว
    เพิ่มเติมลงในคอมพิวเตอร์พกพา ทำให้สามารถประมวลผลและจัดทำเอกสารรูปเล่มโครงงานต่อไปได้อย่างสมบูรณ์
\end{enumerate}

\subsection*{ปัญหาที่พบ แต่ยังแก้ไขไม่ได้}
\begin{enumerate}
    \item การสื่อสารภายในกลุ่มผู้จัดทำที่ไม่สม่ำเสมอ สาเหตุมาจากเวลาว่างของสมาชิกในกลุ่มที่ไม่ตรงกัน และไม่มีการกำหนดความถี่ในการอัปเดตงานร่วมกันอย่างเป็นทางการตั้งแต่แรก 
    ทำให้เกิดความคลาดเคลื่อนในการทำงานและทำความเข้าใจรายละเอียดของงานไม่ตรงกัน ทางผู้จัดทำจึงพยายามแก้ไขโดยการเพิ่มความถี่ในการติดตามความคืบหน้าของงานอย่างใกล้ชิด 
    แต่ปัญหาการสื่อสารภายในกลุ่มผู้จัดทำที่ไม่สม่ำเสมอก็ยังคงมีอยู่ในระดับนึง
    \item เกิดปัญหา LaTeX compile เอกสารใน MacOS ไม่ได้ สาเหตุมาจากข้อผิดพลาดของระบบ LaTeX version 2025 ทางผู้จัดทำได้พยายามแก้ไขปัญหาเบื้องต้นโดยการเปลี่ยนไปใช้ LaTeX version 2024 แทน 
    แต่ก็ยังพบปัญหาในการประมวลผลเอกสารอยู่ดี จึงถือเป็นปัญหาที่ยังแก้ไขไม่ได้ในขณะนี้
\end{enumerate}

\section{\ifenglish Suggestions and further improvements \else ข้อเสนอแนะและแนวทางการพัฒนาต่อ\fi}

\subsection*{ด้านการออกแบบ UX/UI}
\paragraph{ควรมีการพัฒนาระบบเสียง}
โดยเพิ่มทั้งระบบเสียงบรรยาย (voice-over) ประกอบเนื้อหาบทเรียน เพื่อช่วยให้นักเรียนระดับประถมศึกษาเข้าใจเรื่องเศษส่วนได้ดียิ่งขึ้น 
ควบคู่ไปกับการใช้ดนตรีประกอบ (background music) เพื่อดึงดูดความสนใจและสร้างบรรยากาศที่สนุกสนาน ทั้งนี้ ควรออกแบบให้ระบบเสียงทำงานสอดคล้องกัน เช่น 
การลดระดับเสียงดนตรีประกอบลงโดยอัตโนมัติ (audio ducking หรือ fade) เมื่อมีเสียงบรรยาย เพื่อให้ผู้เรียนสามารถจดจ่อกับเนื้อหาได้อย่างเต็มที่โดยไม่ถูกรบกวน 
และเป็นการรองรับรูปแบบการรับรู้ที่หลากหลายของผู้เรียน

\subparagraph{ในมุมมองด้านการออกแบบ}
การพัฒนาระบบเสียงมีความท้าทายและยาก ต้องใช้เวลาในการดำเนินการค่อนข้างสูง เนื่องจากมีความจำเป็นต้องศึกษาและวิจัยข้อมูลเพิ่มเติมเกี่ยวกับความเหมาะสมของโทนเสียง จังหวะ 
และรูปแบบของดนตรีประกอบที่จะส่งผลดีต่อสมาธิและการเรียนรู้ของเด็กระดับประถมศึกษา รวมถึงต้องพิจารณาถึงกระบวนการจัดหาและบันทึกไฟล์เสียงที่มีคุณภาพและถูกต้องตามลิขสิทธิ์

\subparagraph{ในมุมมองด้านการพัฒนา}
การเพิ่มฟังก์ชันดังกล่าวสามารถดำเนินการได้อย่างรวดเร็วและมีความซับซ้อนต่ำ เนื่องจากโครงสร้างของ web application ในปัจจุบันได้ถูกออกแบบมาให้รองรับการจัดการระบบเสียงไว้เรียบร้อยแล้ว 
ผู้พัฒนาระบบจึงสามารถนำไฟล์เสียงที่ผ่านการออกแบบและคัดเลือกแล้ว เข้าไปติดตั้งในระบบและเปิดใช้งานฟังก์ชันดังกล่าวได้ทันที

\paragraph{ควรมีการพัฒนาระบบ UI ที่สามารถปรับแต่งได้}
เพื่อให้ผู้ใช้งานสามารถปรับแต่ง UI ได้ด้วยตนเอง เช่น การแต่งตัวมาสคอต, การเปลี่ยนภาพพื้นหลัง หรือการปรับเปลี่ยนสีธีม เพื่อเพิ่มความน่าสนใจ 
กระตุ้นการมีส่วนร่วม และตอบสนองต่อความชอบส่วนบุคคลของนักเรียน 

\subparagraph{ในมุมมองด้านการออกแบบ}
การพัฒนาระบบ UI ที่สามารถปรับแต่งได้มีความท้าทายและยาก เนื่องจากต้องใช้เวลาในการออกแบบและวาดภาพประกอบเพิ่มเติมสำหรับของตกแต่งใหม่ทั้งหมด 
รวมถึงต้องวาดภาพ mascot ในลักษณะที่สวมใส่ของตกแต่งแต่ละชิ้นแยกกัน เพื่อให้ระบบสามารถนำไปประมวลผลและแสดงภาพได้อย่างสมบูรณ์แบบ 
นอกจากนี้ ยังต้องพิจารณาถึงการออกแบบ UI ที่สามารถรองรับการปรับแต่งได้อย่างยืดหยุ่นและใช้งานง่ายสำหรับเด็กระดับประถมศึกษาอีกด้วย

\subparagraph{ในมุมมองด้านการพัฒนา}
การเพิ่มฟังก์ชันดังกล่าวสามารถดำเนินการได้อย่างรวดเร็วและมีความซับซ้อนต่ำ เนื่องจากโครงสร้างของ web application ในปัจจุบันได้ถูกออกแบบมาให้รองรับการจัดการระบบการปรับแต่ง UI ไว้เรียบร้อยแล้ว 
ผู้พัฒนาระบบจึงสามารถนำไฟล์ภาพประกอบที่ผ่านการออกแบบและวาดเพิ่มเติมแล้ว เข้าไปติดตั้งในระบบและเปิดใช้งานฟังก์ชันดังกล่าวได้ทันที


\subsection*{ด้านการพัฒนา Back-end}
\begin{itemize}
    \item ควรพัฒนาระบบการ synchronize ข้อมูลข้ามอุปกรณ์ เพื่อให้ผู้เรียนสามารถติดตามความคืบหน้าของบทเรียนได้อย่างต่อเนื่อง และสามารถเข้าถึงสื่อการสอนได้จากหลากหลายอุปกรณ์
    \item ควรพัฒนาระบบการจัดการ configuration ที่รองรับการแสดงผลลัพธ์แบบทันที (real-time preview) เพื่อให้ผู้พัฒนาสามารถตรวจสอบการเปลี่ยนแปลงรูปแบบหรือเนื้อหาของสื่อการสอนได้ควบคู่ไปกับการแก้ไข 
    ซึ่งจะช่วยลดข้อผิดพลาดและเพิ่มความสะดวกในการจัดการระบบ
\end{itemize}
