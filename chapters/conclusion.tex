\section{\ifenglish Challenges\else ปัญหาที่พบและแนวทางการแก้ไข\fi}

ในการทำโครงงานนี้ พบว่าเกิดปัญหาหลัก ๆ ดังนี้
\begin{enumerate}
    \item เกิดปัญหาคอขวด (bottleneck) ในขั้นตอนการออกแบบ เนื่องจากในช่วงเริ่มต้นมีผู้รับผิดชอบในส่วนของการออกแบบเพียงคนเดียว 
    ส่งผลให้กระบวนการทำงานในขั้นตอนอื่น ๆ ที่ต้องรอรูปแบบการออกแบบเกิดความล่าช้า ทางผู้จัดทำจึงแก้ไขโดยการเพิ่มจำนวนสมาชิกเข้ามาช่วยรับผิดชอบในส่วนของการออกแบบเพิ่มเติม 
    เพื่อกระจายภาระงานและลดความล่าช้าในขั้นตอนดังกล่าว
    \item การกระจายภาระงานที่ไม่สมดุล ในช่วงแรกของการดำเนินงาน การแบ่งหน้าที่รับผิดชอบของสมาชิกในกลุ่มยังไม่เหมาะสมและไม่สอดคล้องกับปริมาณงานที่เกิดขึ้นจริง 
    ส่งผลให้สมาชิกบางคนมีภาระงานที่หนักเกินไป ทางผู้จัดทำจึงแก้ไขโดยการนำกระบวนการทำงานทั้งหมดมาทบทวนร่วมกันใหม่ 
    และจัดสรรภาระงานให้มีความสมดุลและเหมาะสมกับความถนัดของสมาชิกแต่ละคนมากขึ้น เพื่อให้การดำเนินงานเป็นไปอย่างมีประสิทธิภาพและเสร็จสิ้นตามกำหนดเวลา
    \item การสื่อสารภายในกลุ่มผู้จัดทำที่ไม่สม่ำเสมอ ทำให้เกิดความคลาดเคลื่อนในการทำงานและทำความเข้าใจรายละเอียดของงานไม่ตรงกัน 
    ทางผู้จัดทำจึงแก้ไขโดยการกำหนดตารางการประชุมและติดตามความคืบหน้าของงานร่วมกันอย่างสม่ำเสมอ เพื่ออัปเดตข้อมูลและปรับความเข้าใจในเป้าหมายให้ตรงกัน
    \item การบริหารจัดการเวลาในส่วนของการพัฒนาระบบ ในช่วงแรกของการพัฒนาระบบ เกิดความล่าช้าและมีความเสี่ยงที่จะดำเนินการไม่แล้วเสร็จตามกำหนดเวลา 
    ทางผู้จัดทำจึงแก้ไขปัญหาโดยการนำส่วนประกอบของระบบ (components) ที่ออกแบบและพัฒนาเสร็จแล้วกลับมาใช้ซ้ำในหลาย ๆ ส่วน เพื่อลดความซ้ำซ้อนในการเขียนโค้ดใหม่ 
    ส่งผลให้สามารถลดระยะเวลาการทำงานและพัฒนาระบบได้เสร็จสิ้นตามเป้าหมาย
    \item การสูญหายของเอกสารหนังสือเชิญ ในขั้นตอนการติดต่อประสานงาน ทางโรงเรียนที่เป็นกลุ่มเป้าหมายได้ทำเอกสารหนังสือเชิญสูญหาย 
    ส่งผลให้กระบวนการติดต่อเพื่อขอลงพื้นที่เกิดความล่าช้า ทางผู้จัดทำจึงแก้ไขโดยการติดต่อประสานงานกับเจ้าหน้าที่ธุรการของภาควิชาวิศวกรรมคอมพิวเตอร์ 
    เพื่อขอออกสำเนาหนังสือเชิญอีกครั้ง และนำไปส่งมอบให้แก่ทางโรงเรียนใหม่เพื่อดำเนินการในขั้นต่อไป
\end{enumerate}

\section{\ifenglish Suggestions and further improvements \else ข้อเสนอแนะและแนวทางการพัฒนาต่อ\fi}

\subsection*{ด้านการออกแบบ UX/UI}
\begin{itemize}
    \item ควรมีการเพิ่มระบบเสียงบรรยาย (voice-over) ประกอบเนื้อหาบทเรียน เพื่อช่วยให้นักเรียนระดับประถมศึกษาเข้าใจเนื้อหาเรื่องเศษส่วนได้ดียิ่งขึ้น 
    และเป็นการรองรับรูปแบบการรับรู้ที่หลากหลายของผู้เรียน
    \item ควรมีการเพิ่มดนตรีประกอบ (background music) ในระหว่างการใช้งานสื่อการสอน เพื่อดึงดูดความสนใจและสร้างบรรยากาศที่สนุกสนานซึ่งช่วยส่งเสริมการเรียนรู้
    \item ควรพัฒนาระบบให้ผู้ใช้งานสามารถปรับแต่ง UI ได้ด้วยตนเอง เช่น การเปลี่ยนภาพพื้นหลัง หรือการปรับเปลี่ยนสีธีม (theme color) เพื่อเพิ่มความน่าสนใจ 
    กระตุ้นการมีส่วนร่วม และตอบสนองต่อความชอบส่วนบุคคลของนักเรียน
\end{itemize}

\subsection*{ด้านการพัฒนา Back-end}
\begin{itemize}
    \item ควรพัฒนาระบบการ synchronize ข้อมูลข้ามอุปกรณ์ เพื่อให้ผู้เรียนสามารถติดตามความคืบหน้าของบทเรียนได้อย่างต่อเนื่อง และสามารถเข้าถึงสื่อการสอนได้จากหลากหลายอุปกรณ์
    \item ควรพัฒนาระบบการจัดการ configuration ที่รองรับการแสดงผลลัพธ์แบบทันที (real-time preview) เพื่อให้ผู้พัฒนาสามารถตรวจสอบการเปลี่ยนแปลงรูปแบบหรือเนื้อหาของสื่อการสอนได้ควบคู่ไปกับการแก้ไข 
    ซึ่งจะช่วยลดข้อผิดพลาดและเพิ่มความสะดวกในการจัดการระบบ
\end{itemize}
