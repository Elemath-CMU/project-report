\section{การสอนเนื้อหาเรื่องเศษส่วน}

การสอนเนื้อหาเรื่องเศษส่วนให้กับเด็กนั้น มีหลายวิธีการที่สามารถนำมาใช้ได้ เช่น การใช้ภาพประกอบ การใช้วัตถุจริง โดยสำหรับเด็กที่พึ่งเริ่มเรียนรู้เรื่องเศษส่วนนั้น การใช้ Fraction Model ที่เป็น Length Model จะช่วยให้เด็กเข้าใจแนวคิดของเศษส่วนได้ง่ายมากกว่า Model อื่นๆ

\section{ความยากของเนื้อหาเรื่องเศษส่วน}

\section{การออกแบบโครงสร้างข้อมูลเกม (Game Configuration Documentation)}

ในส่วนนี้จะอธิบายโครงสร้างข้อมูลของระบบเกมที่ใช้ในสื่อการสอน โดยมีวัตถุประสงค์เพื่อกำหนดรูปแบบข้อมูลที่ใช้ควบคุมลำดับเหตุการณ์ (game flow) และการโต้ตอบ (interaction) ภายในเกมอย่างเป็นระบบ

\subsection{ภาพรวมของโครงสร้างข้อมูลเกม}

โครงสร้างของเกมถูกออกแบบในรูปแบบของ ``ตอน'' (Episode) โดยแต่ละตอนประกอบด้วยลำดับของการโต้ตอบ (Interaction) ที่ผู้เล่นจะต้องดำเนินการตามลำดับ

โดยสามารถสรุปโครงสร้างได้ดังนี้

\begin{itemize}
    \item เกมประกอบด้วยหลาย Episode
    \item แต่ละ Episode เป็นรายการ (array) ของ Interaction
    \item Interaction แต่ละตัวมีประเภทที่แตกต่างกันตามลักษณะการใช้งาน
\end{itemize}

โครงสร้างข้อมูลหลักอยู่ในรูปแบบ

\begin{itemize}
    \item gameData $\rightarrow$ รายการของ Episode ทั้งหมด
    \item Episode $\rightarrow$ รายการของ Interaction
    \item Interaction $\rightarrow$ หน่วยย่อยของการเล่นในแต่ละขั้น
\end{itemize}

\subsection{ประเภทของการโต้ตอบ (Interaction Types)}

Interaction ภายในเกมแบ่งออกเป็น 3 ประเภทหลัก ได้แก่

\begin{enumerate}
    \item \textbf{Playground} \\
    เป็นการโต้ตอบหลักของเกม ผู้เล่นสามารถลากวางวัตถุ และระบบจะตรวจสอบคำตอบตามเงื่อนไขที่กำหนด

    \item \textbf{Choice Answer} \\
    เป็นการเลือกคำตอบแบบหลายตัวเลือก โดยแต่ละตัวเลือกจะนำไปสู่ Interaction ถัดไปที่แตกต่างกัน

    \item \textbf{Checkpoint} \\
    เป็นขั้นตอนแสดงข้อความเพื่อสื่อสารเนื้อหา โดยไม่มีการโต้ตอบจากผู้เล่น
\end{enumerate}

\subsection{โครงสร้างบทสนทนา (Dialogue)}

ใน Interaction ประเภท Playground จะมีระบบบทสนทนาเพื่อแนะนำผู้เล่น โดยแต่ละข้อความประกอบด้วย

\begin{itemize}
    \item ข้อความที่แสดง
    \item สถานะว่าสามารถกดข้ามได้หรือไม่
\end{itemize}

ซึ่งช่วยควบคุมจังหวะการเรียนรู้ของผู้เล่นให้เป็นลำดับขั้น

\subsection{การออกแบบวัตถุภายในเกม (Object Model)}

ภายในแต่ละ Interaction สามารถมีวัตถุ (Object) หลายประเภทเพื่อใช้ในการเรียนรู้และโต้ตอบ โดยวัตถุแต่ละชนิดมีคุณสมบัติพื้นฐานร่วมกัน เช่น

\begin{itemize}
    \item รหัสวัตถุ (id)
    \item ตำแหน่ง (x, y)
    \item สถานะการเคลื่อนที่ (fixed)
\end{itemize}

ประเภทของวัตถุหลัก ได้แก่

\begin{enumerate}
    \item \textbf{Pencil} ใช้แทนหน่วยความยาว มีคุณสมบัติด้านสี ขนาด และทิศทาง
    \item \textbf{Stick} ใช้แทนแท่งความยาวที่สามารถถูกตัดได้
    \item \textbf{Other Object} วัตถุทั่วไปที่กำหนดรูปแบบด้วย SVG
    \item \textbf{Spawner} วัตถุที่ใช้สร้างวัตถุใหม่แบบลากออกมาได้
    \item \textbf{Fraction Input} ช่องสำหรับกรอกค่าของเศษส่วน (ตัวเศษ/ตัวส่วน)
    \item \textbf{Message} วัตถุแสดงข้อความในฉาก (ใช้เพื่อช่วยอธิบาย)
\end{enumerate}

\subsection{ระบบตรวจสอบคำตอบ (Rule System)}

ระบบ Rule ใช้สำหรับตรวจสอบว่าการกระทำของผู้เล่นถูกต้องหรือไม่ และกำหนดการเปลี่ยนไปยัง Interaction ถัดไป

ประเภทของ Rule ที่สำคัญ ได้แก่

\begin{itemize}
    \item การลากวัตถุไปวางบนวัตถุอื่น
    \item การลากไปยังพื้นที่ที่กำหนด
    \item การจัดตำแหน่งให้ตรงจุด
    \item การกรอกค่าเศษส่วน
    \item การจบบทสนทนา
\end{itemize}

Rule แต่ละตัวจะกำหนด

\begin{itemize}
    \item เงื่อนไขของคำตอบ
    \item Interaction ถัดไปเมื่อเงื่อนไขสำเร็จ
\end{itemize}

นอกจากนี้ยังสามารถกำหนดหลาย Rule ใน Interaction เดียว เพื่อรองรับสถานการณ์ที่ซับซ้อน

\subsection{โครงสร้างของ Interaction แบบ Playground}

Interaction ประเภท Playground ประกอบด้วยองค์ประกอบหลัก ได้แก่

\begin{itemize}
    \item บทสนทนา (Dialogues)
    \item วัตถุในฉาก (Objects)
    \item กฎการตรวจคำตอบ (Rules)
\end{itemize}

โดยผู้เล่นจะต้องทำการโต้ตอบกับวัตถุจนกว่าจะผ่านเงื่อนไขที่กำหนด

\subsection{โครงสร้างของ Interaction แบบ Choice Answer}

ประกอบด้วย

\begin{itemize}
    \item ข้อความคำถาม
    \item ตัวเลือกคำตอบ
    \item การกำหนดเส้นทางไปยัง Interaction ถัดไป
\end{itemize}

เหมาะสำหรับการวัดความเข้าใจเชิงแนวคิด

\subsection{ผลลัพธ์ของการตรวจสอบคำตอบ}

เมื่อผู้เล่นทำการโต้ตอบ ระบบจะประมวลผลและส่งผลลัพธ์ เช่น

\begin{itemize}
    \item มีการเปลี่ยนแปลงหรือไม่
    \item วัตถุที่ต้องลบออก
    \item วัตถุที่ต้องย้ายตำแหน่ง
    \item Interaction ถัดไป
\end{itemize}

ซึ่งช่วยให้ระบบสามารถควบคุมสถานะของเกมได้อย่างยืดหยุ่น

\subsection{แนวทางในการออกแบบข้อมูล (Authoring Guidelines)}

เพื่อให้การพัฒนา Episode เป็นไปอย่างถูกต้อง ควรปฏิบัติตามแนวทางดังนี้

\begin{enumerate}
    \item กำหนดหมายเลข Interaction ไม่ซ้ำกัน
    \item ตรวจสอบให้ทุก nextInteraction อ้างอิงถึงขั้นตอนที่มีอยู่จริง
    \item ใช้ id ของวัตถุให้คงที่เพื่อรองรับการตรวจสอบ
    \item ใช้ fixed กับวัตถุที่ไม่ต้องการให้เคลื่อนที่
    \item แยก Interaction เป็นขั้นตอนย่อยเพื่อลดความซับซ้อน
\end{enumerate}

\subsection{หมายเหตุเชิงเทคนิค}

ในปัจจุบันมีการกำหนดประเภทของวัตถุ Message ไว้แล้ว แต่ยังไม่ได้รวมในกลุ่ม ObjectData อย่างเป็นทางการ ซึ่งสามารถปรับปรุงเพิ่มเติมได้ในอนาคตเพื่อรองรับการใช้งานอย่างสมบูรณ์

จากการออกแบบดังกล่าว ทำให้ระบบเกมมีความยืดหยุ่นสูง สามารถเพิ่มเนื้อหาใหม่ได้ง่าย และรองรับการออกแบบการเรียนรู้แบบเป็นลำดับขั้น (progressive learning) ได้อย่างมีประสิทธิภาพ
